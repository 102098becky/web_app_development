# PRD (Product Requirements Document) - 個人記帳簿系統

## 1. 專案概述

### 背景與動機
現代人生活忙碌，常常不清楚自己的錢花在哪裡，導致不易累積財富或發生超支狀況。市面上的記帳軟體雖然多，但有時過於複雜。因此我們希望能打造一款介面簡單直覺、功能實用的「個人記帳簿系統」，幫助使用者輕鬆記錄每日收支，並透過分類與統計功能，掌握自身的財務狀況，達成理財目標。

### 目標用戶
- 有理財與儲蓄習慣的學生或上班族。
- 需要管理家庭開銷的家庭主婦／主夫。
- 想要清楚追蹤每一筆收入與支出，但不想花太多時間學習複雜系統的人。

### 核心價值主張
- **簡單直覺**：操作流程極簡化，讓記帳成為每天順手的習慣。
- **清晰統計**：即時掌握餘額與收支分佈，財務狀況一目瞭然。
- **多帳本管理**：可根據不同用途分開記帳（如日常開銷、旅遊基金等）。

---

## 2. 功能需求

以下列出系統的主要功能，並以使用者故事（User Story）的格式描述：

1. **記錄收入與支出**
   - 作為 **一般使用者**，我希望 **能夠輸入每一筆的收入與支出金額、日期及備註**，以便 **記錄我每天的財務進出**。

2. **顯示餘額統計**
   - 作為 **一般使用者**，我希望 **可以在首頁或儀表板看到目前的總收支與結餘狀況**，以便 **隨時掌握我還剩多少錢可以運用**。

3. **分類收支**
   - 作為 **一般使用者**，我希望 **在記帳時能選擇預設的分類（如餐飲、交通、薪水等），或自訂分類**，以便 **日後能分析各項支出的比例**。

4. **新增記帳本**
   - 作為 **一般使用者**，我希望 **能夠建立並切換多個不同的記帳本（但有創建數量限制，例如上限 3 至 5 本）**，以便 **將生活費、旅行公費等不同來源的帳目分開管理，避免混淆**。

5. **導出紀錄**
   - 作為 **一般使用者**，我希望 **可以將指定區間的記帳紀錄導出為 CSV 或 Excel 檔案**，以便 **使用其他軟體進行更進階的分析或備份資料**。

---

## 3. 非功能需求

### 技術限制
- **後端框架**：使用 Flask 進行輕量級網站開發。
- **前端渲染**：使用 Jinja2 模板引擎搭配基本的 HTML/CSS/JavaScript。
- **資料庫**：使用 SQLite 作為關聯式資料庫，方便輕量佈署與本地儲存。

### 效能與安全考量
- **效能**：系統操作回饋時間不應超過 1 秒，保持記帳過程順暢。
- **安全性**：
  - 若未來支援多用戶，應實作使用者驗證（Authentication）與密碼加密。
  - 對於所有的表單輸入（金額、備註等），需做好防範 SQL Injection 與 XSS 攻擊的過濾措施。

## 4. MVP (Minimum Viable Product) 範圍

為了快速驗證並發佈第一版，我們將功能分為三個優先層級：

| 優先級別 | 功能項目 | 包含內容 |
| --- | --- | --- |
| **Must Have** (必備) | 核心記帳與統計 | - 記錄單筆收入與支出<br>- 顯示當月餘額與總收支<br>- 基本的收支分類選擇器 |
| **Should Have** (應備) | 多帳本管理 | - 新增、刪除記帳本<br>- 切換不同記帳本的功能 |
| **Nice to Have** (選備) | 進階功能與導出 | - 自訂分類項目名稱<br>- 將記帳明細導出為 CSV 格式 |
